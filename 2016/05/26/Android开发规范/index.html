Andriod 开发规范

  • Android
  • 总结
Android

“The only way to make the deadline - the only way to go fast - is to keep the code as clean as possible at all times.” –Robert C. Martin

一、书写规范

1.编码

编码方式统一用UTF-8. Android Studio默认已是UTF-8。

2.缩进

缩进统一为4个空格，将Tab size设置为4则可以保证tab键按4个空格缩进。另外，不要勾选上Use tab character，可以保证切换到不同tab长度的环境时还能继续保持统一的4个空格的缩进样式。

3.括号

花括号不要单独一行，和它前面的代码同一行。而且，花括号与前面的代码之间用一个空格隔开。

4.空格

if、else、for、switch、while等逻辑关键字与后面的语句留一个空格隔开。

运算符两边各用一个空格隔开。

方法的每个参数之间用一个空格隔开。

5.空行

将逻辑相关的代码段用空行隔开，以提高可读性。空行也只空一行，不要空多行。在以下情况需用一个空行：

  • 两个方法之间
  • 方法内的两个逻辑段之间
  • 方法内的局部变量和方法的第一条逻辑语句之间（一般用不着，用得着的话可能需要重构了）
  • 常量和变量之间

6.换行

当一个表达式无法容纳在一行内时，可换行显示，另起的新行用8个空格缩进。

7.声明

一行声明一个变量，不要一行声明多个变量，这样有利于写注释。

8.行宽

行宽设置为100，设置格式化时自动断行到行宽位置。

9.格式化

使用快捷键进行代码自动格式化，每次提交前请右键单击项目文件夹–>Reformat Code–>勾选上Optimize imports和File masks(*.java)–>Run

10.方法行数

一个方法最多不要超过40行代码，大概一个屏幕

11.常量

范围型的常量用枚举类定义，而不要直接用整型或字符，这样可以减少范围值的有效性检查。

12.大小单位

文字大小的单位统一用sp，元素大小的单位统一用dp。

13.String值定义

应用中的字符串统一在strings.xml中定义，然后在代码和布局文件中引用。

14.颜色值定义

颜色值统一在colors.xml中定义，然后在代码和布局文件中引用。另外，不要在代码和布局文件中引用系统的颜色，除了透明，因为一旦升级版本，可能会改变。

命名规范

1.包

域名反写+项目名称+模块名称，全部单词用小写字母。
例如，Grab项目的app模块包名如下：
cn.romens.grab.app

2.类和接口

使用大驼峰规则，用名词或名词词组命名，每个单词的首字母大写。
以下为几种常用类的命名：

  • activity类，命名以Activity为后缀，如：LoginActivity
  • fragment类，命名以Fragment为后缀，如：ShareDialogFragment
  • service类，命名以Service为后缀，如：DownloadService
  • adapter类，命名以Adapter为后缀，如：CouponListAdapter
  • 工具类，写在util包下
  • 接口实现类，如果是单一实现类，命名以Impl为后缀，如：ApiImpl

3.方法

使用小驼峰规则，用动词命名，第一个单词的首字母小写，其他单词的首字母大写。
以下为几种常用方法的命名：

  • 初始化方法，命名以init开头，例：initView
  • 流程的下一步方法，命名以to开头，例：toLogin
  • 设置方法，命名以set开头，例：setData
  • 具有返回值的获取方法，命名以get开头，例：getData
  • 通过异步加载数据的方法，命名以load开头，例：loadData
  • 布尔型的判断方法，命名以is或has，或具有逻辑意义的单词如equals，例：isEmpty

4. 控件缩写

控件 缩写 控件 缩写
TextView tv EditText edit
Button btn ImageButton imgbtn
ImageView img ListView listview
RadioGroup group RadioButton radiobtn
ProgressBar progress SeekBar seek
CheckBox chk Spinner spinner
TableLayout table TableRow row
LinearLayout ll RelativeLayout rl
SearchView search TabWidget widget

5.常量

全部为大写单词，单词之间用下划线分开。

6.变量

如果变量较少，直接使用意义描述即可；如果变量较多，容易重复的话，使用{范围描述+}意义描述+类型描述的组合，用驼峰式，首字母小写。

7.控件id

如果控件较少，直接使用意义描述即可；如果变量较多，容易重复的话，使用{范围}意义控件缩写，范围可选，只在有明确定义的范围内才需要加上。

8.layout

组件类型{范围}功能，范围可选，只在有明确定义的范围内才需要加上。
以下为几种常用的组件类型命名：

  • activity{范围}功能，为Activity的命名格式
  • fragment{范围}功能，为Fragment的命名格式
  • dialog{范围}功能，为Dialog的命名格式
  • itemlist{范围_}功能，为ListView的item命名格式
  • itemrecycler{范围_}功能，为RecyclerView的item命名格式
  • itemgrid{范围_}功能，为GridView的item命名格式
  • headerlist{范围_}功能，为ListView的HeaderView命名格式
  • footerlist{范围_}功能，为ListView的FooterView命名格式
  • include_功能，用于公共布局（引用方式：include）
  • custom功能类型，自定义控件布局

9.strings

范围_功能，范围可选，以功能分类写注释
以下为几种常用的命名：

  • LoginActivity中的页面标题，命名格式为：login_header_title
  • LoginActivity中的按钮文字，命名格式为：loginbtn按钮事件
  • LoginFragment中的标签文字，命名格式为：loginlabel标签文字
  • 选项卡文字，命名格式为：范围(tab)选项卡文字
  • 消息框文字，命名格式为：范围(toast)消息
  • 编辑框的提示文字，命名格式为：范围hint提示信息
  • 图片的描述文字，命名格式为：范围(desc)图片文字
  • 对话框的文字，命名格式为：范围dialog文字
  • menu的item文字，命名格式为：范围action文字

10.colors

以颜色命名，写了不能改

11.drawable

前缀{控件}{形状/颜色/etc.}{_后缀}，控件、后缀可选，但控件和范围至少要有一个。

  • 图标类，添加ic前缀
  • 背景类，添加bg前缀
  • 分隔类，添加div前缀
  • 区分状态时，默认状态，添加normal后缀
  • 区分状态时，按下时的状态，添加pressed后缀
  • 区分状态时，选中时的状态，添加selected后缀
  • 区分状态时，不可用时的状态，添加disable后缀
  • 多种状态的，添加selector后缀（一般为ListView的selector或按钮的selector）

12.动画文件

动画类型_动画方向。

  • fade_in，淡入
  • fade_out，淡出
  • push_down_in，从下方推入
  • push_down_out，从下方推出
  • slide_in_from_top，从头部滑动进入
  • zoom_enter，变形进入
  • shrink_to_middle，中间缩小

注释规范

必要时写注释

命名写清晰

注释就用不着了，除非你自己都看不懂了
